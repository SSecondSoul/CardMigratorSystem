# 卡片组件复杂度五维评估体系

本文档定义了本研究中卡片组件复杂度的五个核心维度及评估标准，用于对组件进行分级（简单/中等/复杂）。

## 1. 五维评估框架

| 维度 | 代码要素 | 评估指标 | 权重 |
|------|----------|----------|------|
| **T: 模板结构** | `<template>` | 节点数、嵌套深度、指令数量 | 1-3分 |
| **D: 数据逻辑** | `data`, `computed`, `methods` | 字段数、计算属性数、方法复杂度 | 1-3分 |
| **I: 交互行为** | `@click`, `$emit`, 生命周期 | 事件数、自定义事件、钩子 | 1-3分 |
| **S: 样式特征** | `<style>`, `:class` | 类数量、动态类、作用域 | 1-3分 |
| **C: 组件通信** | `props`, `$emit`, 子组件 | props数、emit事件、子组件数 | 1-3分 |

**总分范围**: 5-15分

---

## 2. 详细评分标准

### 2.1 模板结构 (T)

评估组件的DOM结构和指令使用情况。

| 分值 | 判定标准 | 示例 |
|------|----------|------|
| **1分** (简单) | • 节点数 ≤ 10<br>• 嵌套深度 ≤ 3层<br>• 指令数 ≤ 2个<br>• 无 `v-for`/`v-if` 嵌套 | 天气卡片：`<div>` → `<h3>`+`<p>` |
| **2分** (中等) | • 节点数 11-30<br>• 嵌套深度 4-5层<br>• 指令数 3-5个<br>• 有单层 `v-for` 或 `v-if` | 待办卡片：列表渲染 |
| **3分** (复杂) | • 节点数 ≥ 31<br>• 嵌套深度 ≥ 6层<br>• 指令数 ≥ 6个<br>• 多层 `v-for`/`v-if` 嵌套 | 仪表盘卡片：多级嵌套列表 |

### 2.2 数据逻辑 (D)

评估组件的数据管理和业务逻辑复杂度。

| 分值 | 判定标准 | 示例 |
|------|----------|------|
| **1分** (简单) | • data字段 ≤ 3个<br>• 无 computed<br>• methods ≤ 2个<br>• 无异步操作 | 天气卡片：`temp`, `city` |
| **2分** (中等) | • data字段 4-8个<br>• computed 1-2个<br>• methods 3-5个<br>• 有简单逻辑 | 待办卡片：增删改 |
| **3分** (复杂) | • data字段 ≥ 9个<br>• computed ≥ 3个<br>• methods ≥ 6个<br>• 有异步/复杂逻辑 | 仪表盘：数据聚合计算 |

### 2.3 交互行为 (I)

评估组件的用户交互和事件处理复杂度。

| 分值 | 判定标准 | 示例 |
|------|----------|------|
| **1分** (简单) | • 事件数 ≤ 1个<br>• 无自定义事件<br>• 无生命周期钩子 | 天气卡片：点击展开 |
| **2分** (中等) | • 事件数 2-4个<br>• 有自定义事件<br>• 1个生命周期钩子 | 待办卡片：增删改+通知父组件 |
| **3分** (复杂) | • 事件数 ≥ 5个<br>• 多级自定义事件<br>• ≥ 2个生命周期钩子 | 仪表盘：数据加载、刷新、销毁 |

### 2.4 样式特征 (S)

评估组件的样式复杂度和动态性。

| 分值 | 判定标准 | 示例 |
|------|----------|------|
| **1分** (简单) | • 类名 ≤ 5个<br>• 无动态类名<br>• 无作用域样式 | 天气卡片：静态样式 |
| **2分** (中等) | • 类名 6-15个<br>• 简单动态类 (1-2个)<br>• 有 scoped | 待办卡片：完成状态样式 |
| **3分** (复杂) | • 类名 ≥ 16个<br>• 复杂动态类 (≥3个)<br>• 有主题切换 | 仪表盘：多主题动态样式 |

### 2.5 组件通信 (C)

评估组件与外部世界的交互复杂度。

| 分值 | 判定标准 | 示例 |
|------|----------|------|
| **1分** (简单) | • props ≤ 2个<br>• 无 emit<br>• 无子组件 | 天气卡片：纯展示 |
| **2分** (中等) | • props 3-5个<br>• emit 1-2个<br>• 子组件 ≤ 1个 | 待办卡片：通知父组件 |
| **3分** (复杂) | • props ≥ 6个<br>• emit ≥ 3个<br>• 子组件 ≥ 2个 | 仪表盘：多子组件协作 |

---

## 3. 复杂度分级

根据总分将组件分为三个等级：

| 等级 | 总分范围 | 特征 | 迁移难度 | 典型组件 |
|------|----------|------|----------|----------|
| **简单** | 5-8分 | 扁平结构、基础数据、少量交互 | 低 | 天气卡片、股票卡片 |
| **中等** | 9-12分 | 嵌套结构、计算属性、多个事件 | 中 | 待办卡片、登录表单 |
| **复杂** | 13-15分 | 深层嵌套、复杂逻辑、多组件通信 | 高 | 仪表盘、图表卡片 |
